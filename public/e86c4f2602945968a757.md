## GitBucket 4.45.0 リリース

- [GitBucket 4.45.0をリリースしました - たけぞう瀕死ブログ](https://takezoe.hatenablog.com/entry/2026/01/10/132525)
- [GitBucket 4.45.0 released!](https://gitbucket.github.io/gitbucket-news/gitbucket/2026/01/10/gitbucket-4.45.0.html)

[GitBucket 4.45.0](https://github.com/gitbucket/gitbucket/releases/tag/4.45.0) が 2026/01/10 にリリースされ、[GitBucket Markdown Enhanced Plugin](https://github.com/yasumichi/gitbucket-markdown-enhanced) が、以下の機能にも適用されるようになりました。

- Commit comments
- Issues
- Pull requests
- Wiki

## GitBucket 4.45.0 の変更点

GitBucket 4.45.0 に関する issue および Pull requests は、[Issues](https://github.com/gitbucket/gitbucket/issues?q=is%3Aclosed+milestone%3A4.45.0) によると以下の通りです。(順序を逆転しています。)

1. [Always use "display name" in the front end instead "username" for better ldap integration](https://github.com/gitbucket/gitbucket/issues/336)
1. [Wiki internal link between pages are broken](https://github.com/gitbucket/gitbucket/issues/2456)
1. [Add support for an alternative renderer to commit comments, wikis, and issues.](https://github.com/gitbucket/gitbucket/pull/3882)
1. [Tentative fix for issue #2456](https://github.com/gitbucket/gitbucket/pull/3883)
1. [Rendering new files that have not been committed](https://github.com/gitbucket/gitbucket/issues/3887)
1. [Preview based on the correct filename on the editor](https://github.com/gitbucket/gitbucket/pull/3894)
1. [Include path in filePath param for preview in the online editor](https://github.com/gitbucket/gitbucket/pull/3896)
1. [Fix typo in #3896](https://github.com/gitbucket/gitbucket/pull/3898)
1. [Remove text-align: left from the cells of a table created with markdown](https://github.com/gitbucket/gitbucket/pull/3899)
1. [Fix system menu of plugins.](https://github.com/gitbucket/gitbucket/pull/3902)
1. [#3894 does not work if deployed with context path](https://github.com/gitbucket/gitbucket/issues/3905)
1. [Fix bug that #3894 doesn't work with context path](https://github.com/gitbucket/gitbucket/pull/3906)
1. [Check existence of Wiki page for Wiki links](https://github.com/gitbucket/gitbucket/pull/3907)
1. [Add option to display fullname instead of username on UI](https://github.com/gitbucket/gitbucket/pull/3910)
1. [Deprecate helpers.markdown()](https://github.com/gitbucket/gitbucket/pull/3913)
1. [Fix warnings in PluginRegistry](https://github.com/gitbucket/gitbucket/pull/3914)
1. [Update gitbucket-gist-plugin for GitBucket 4.45.0](https://github.com/gitbucket/gitbucket/issues/3915)

この内、1, 14 番が、たけぞうさんブログで紹介されていた「ユーザ名にFullnameを表示するオプションを追加」に対応します。

- [Always use "display name" in the front end instead "username" for better ldap integration](https://github.com/gitbucket/gitbucket/issues/336)(1)
- [Add option to display fullname instead of username on UI](https://github.com/gitbucket/gitbucket/pull/3910)(14)

また、3番の [Add support for an alternative renderer to commit comments, wikis, and issues.](https://github.com/gitbucket/gitbucket/pull/3882) が、「MarkdownレンダラプラグインをIssues等にも適用」に対応します。

これに伴い、15番の [Deprecate helpers.markdown()](https://github.com/gitbucket/gitbucket/pull/3913) で使用されなくなった `helpers.markdown()` を Deprecate するなどの微調整が行われたようです。

### 標準マークダウンレンダラーの Wiki リンクがユーザー期待通り動作していなかったのを修正

- [Wiki internal link between pages are broken](https://github.com/gitbucket/gitbucket/issues/2456)(2)
- [Tentative fix for issue #2456](https://github.com/gitbucket/gitbucket/pull/3883)(4)
- [Check existence of Wiki page for Wiki links](https://github.com/gitbucket/gitbucket/pull/3907)(13)

これまで `[[ページ名]]` のような記法で生成される Wiki リンクが、ユーザーの期待通りの URL にならなかったのを修正しています。

元々、Wiki リポジトリ内にある画像を表示するための URL に解決されていたため、暫定対処として、[Tentative fix for issue #2456](https://github.com/gitbucket/gitbucket/pull/3883) で `.` が含まれていなかったら、Wiki ページへのリンクに解決されるように修正しました。

しかし、これでは Wiki ページ名に `.` が含まれている場合に正しく機能しないため、[Check existence of Wiki page for Wiki links](https://github.com/gitbucket/gitbucket/pull/3907) にて、Wiki ページ名が存在するか否かでリンクの解決を切り替えるように修正しました。

### ファイル新規作成時のプレビュー時も対応するレンダラーが使用されるように修正

- [Rendering new files that have not been committed](https://github.com/gitbucket/gitbucket/issues/3887)(5)
- [Preview based on the correct filename on the editor](https://github.com/gitbucket/gitbucket/pull/3894)(6)
- [Include path in filePath param for preview in the online editor](https://github.com/gitbucket/gitbucket/pull/3896)(7)
- [Fix typo in #3896](https://github.com/gitbucket/gitbucket/pull/3898)(8)

GitBucket 4.44.0 以前では、ファイルの新規作成時にプレビューを行った場合、ファイルの種類に関わらず、Plain Text として表示されていました。(プレビューの役目を果たしていませんでした。)

上記修正により、GitBucket 4.45.0 では、新規作成時もファイルの拡張子に応じたプレビューができるようになりました。

ただし、GitBucket 起動時に `--prefix` オプションを指定してコンテキストパスを変更したり、Apache Tomcat などにデプロイするとプレビュー自体が行えないバグがありましたので以下の issue と Pull Requests で修正しました。

- [#3894 does not work if deployed with context path](https://github.com/gitbucket/gitbucket/issues/3905)(11)
- [Fix bug that #3894 doesn't work with context path](https://github.com/gitbucket/gitbucket/pull/3906)(12)

### 表のセルスタイルに text-align: left が指定されていたのを削除

- [Remove text-align: left from the cells of a table created with markdown](https://github.com/gitbucket/gitbucket/pull/3899)(9)

GitBucket のスタイルシート [src/main/webapp/assets/common/css/gitbucket.css](https://github.com/gitbucket/gitbucket/blob/4.44.0/src/main/webapp/assets/common/css/gitbucket.css) において、

```css
div.markdown-body table th,
div.markdown-body table td {
  padding: 8px;
  line-height: 20px;
  text-align: left;
  vertical-align: top;
  border-top: 1px solid #dddddd;
}
```

のように表のセルに `text-align: left;` というスタイルが指定されていたのを削除しました。

これまで `<td align="center">` のように指定してもスタイルシートが優先されるため、必ず左揃えになっていましたが、この修正により、解消しました。

### システム設定においてプラグインで追加したメニューのスタイルが標準機能のスタイルと異なっていたのを修正

- [Fix system menu of plugins.](https://github.com/gitbucket/gitbucket/pull/3902)(10)

これまでプラグイン独自のシステム設定メニューのスタイルが、標準機能のスタイルと異なっていたため、サイドバーのメニューが選択状態(active)になりませんでした。

上記 Pull Request で修正したことにより、プラグイン独自のシステム設定メニューも選択状態(active)になるようになりました。

### gitbucket-gist-plugin を GitBucket 4.45.0 に合わせて更新

- [Update gitbucket-gist-plugin for GitBucket 4.45.0](https://github.com/gitbucket/gitbucket/issues/3915)(17)

[gitbucket-gist-plugin](https://github.com/gitbucket/gitbucket-gist-plugin) 内で GitBucket 4.45.0 で Deprecate になった `helpers.markdown()` を使用していたため、 `helpers.renderMarkup()` を使用するなど、調整を行ったようです。

詳細を知りたい場合は、以下のコミットを確認してください。
