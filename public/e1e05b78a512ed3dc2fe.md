## 概要

[GitBucket](https://gitbucket.github.io/)用のプラグイン [GitBucket Markdown Enhanced Plugin](https://github.com/yasumichi/gitbucket-markdown-enhanced)を開発しています。

GitBucket Markdown Enhanced Plugin は、GitBucket 標準のマークダウンレンダリングエンジンを置き換えるプラグインです。

目標は、[Visual Studio Code](https://code.visualstudio.com/) の [Markdown Preview Enhanced](https://shd101wyy.github.io/markdown-preview-enhanced/#/) 向けの markdown ファイルを軽易に Web で共有できる環境です。

今回は、Yaml frontmatter を使えるようにして、リポジトリのファイルビューアーでスライドリストを表示できるようにしたお話です。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/36738/e8406659-eda7-4407-b936-f90fcb3b0edc.png)

:::note warn
各スライドの大きさは、表示時の幅に依存します
現状では、スライドショーでのレイアウトなどを完全に再現できてはいません。
:::

## Yaml frontmatter の記法

以下のように Yaml frontmatter を記述するとスライドリストを表示することができます。

```Yaml
---
presentation: true
---
```

[Markdown Preview Enhanced の記法](https://shd101wyy.github.io/markdown-preview-enhanced/#/presentation?id=presentation-front-matter) も使えますが、まだ、細かい制御はできません。

現在のところ、トップレベル項目に `presentation` が存在するかを確認しているだけです。

前項の修正に伴い、markdown ソースを取得し、Yaml frontmatter を削除する処理を加えました。

### Introduce slide list view

#### build.sbt の修正

Scala で Yaml を解釈できるようにするために [circe-yaml](https://github.com/circe/circe-yaml) を依存ライブラリを追加しました。

```Scala
libraryDependencies += "io.circe" %% "circe-yaml-v12" % "0.16.0"
```

#### src/main/resources/gme/assets/gme.js の修正

[reveal.js](https://revealjs.com/) が読み込まれている場合は、KaTeX による数式のレンダリングを reveal.js に任せるように修正しました。

#### src/main/resources/gme/assets/slidelist.js を新規作成

スライドリストをレンダリングする JavaScript を新規作成しました。

```JavaScript
(function () {
    let markdownbody = document.querySelector(".markdown-body");
    let slideWidth = Math.trunc(markdownbody.clientWidth / 100) * 100;
    let slideHeight = Math.trunc(slideWidth * 9 / 16);
    document.querySelectorAll('.reveal').forEach(async (n, i) => {
        n.style.width = `${slideWidth}px`;
        n.style.height = `${slideHeight}px`;
        n.style.border = "1px solid gray";
        let deck = new Reveal(n, {
            embedded: true,
            controls: false,
            plugins: [RevealMarkdown, RevealHighlight, RevealMath.KaTeX],
            katex: {
                local: katexUrl
            },
            markdown: {
                gfm: true,
                renderer: gme.renderer,
                tokenizer: gme.tokenizer
            }
        });
        deck.initialize().then(gme.revealReady);
    });
})();
```

#### src/main/scala/io/github/yasumichi/gme/MarkdownEnhancedRenderer.scala の修正

- markdown ファイルから、Yaml frontmatter を取り出す
- Yaml frontmatter のトップレベル項目に `presentation` が存在するか確認
  - frontmatter が存在しない場合や トップレベル項目に `presentation` が存在しない場合は、従来のレンダリングを行う。
  - トップレベル項目に `presentation` が存在した場合
    - reveal.js 等、必要な script タグを出力
    - `---` で markdown を分割し、それぞれを以下の HTML で出力

```HTML
<div class="reveal">
  <div class="slides">
    <section data-markdown>ここに各パートが入る</section>
  </div>
</div>
```

#### src/main/scala/io/github/yasumichi/gme/MarkdownPreprocessor.scala の作成

markdown のプリプロセッサを作成しました。現状では、Yaml frontmatter と markdown body の分離のみを行っています。
