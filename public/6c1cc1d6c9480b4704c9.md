Specifically, the `stickyTop` variable was defined with `position: 'sticky'` and an absolute `top` value, but some components were overriding this with `...stickyTop` spread, which might have conflicted with other styles in `Timetable.css` or the DOM rendering order.

I have updated `src/components/Timetable.tsx` to:
1.  Derive a consistent `resourceStickyTop` string (`headerHeight + eventRowHeight`).
2.  Explicitly apply `position: 'sticky', top: resourceStickyTop` to both the `.grid-label` and the `.grid-cell` elements within the resource rows, ensuring the sticky behavior is correctly inherited and applied.

These changes ensure the sticky headers remain fixed correctly while scrolling.
